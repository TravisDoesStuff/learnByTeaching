Questions:

  • What are the ::before and ::after pseudo-elements in CSS?
    • First remember pseudo-classes pertain to a certain state of a element such as hover, visited, or focus.
    • Pseudo-elements are used to style a specific part of an element.
      • element::before adds content before the content of the element
      • element::after does the same thing but after the element's content
      • use 'content: words' to add content
  • What are the fundamentals of HTTP? Such as the header.
    • HTTP stands for HyperText Transfer Protocol
    • Request methods:
      • GET: retrieve information from server, parameters visible in URL
      • POST: sends information to server
      • PUT: same as POST but uses the same resource, for example when the browser refreshes, POST methods ask about resending the data
      • DELETE: removes the specific resource, infrequently used
      • HEAD: same as GET but without a response body, infrequently used
    • Headers are simply additional data sent in the request, often used for caching , authentication, and cookies.
    • Status Codes:
      • 1xx: Information
      • 2xx: Success
      • 3xx: Redirects
      • 4xx: Client error
      • 5xx: Server error
    • cURL is a HTTP command line tool.
  • What is the DOM?
    • Before reading: It is the representation of all content on the page. Not quite.
    • After reading: DOM is the Document Object Model.
    • An interface between the browser and the HTML/XML. Represented as a hierarchy tree. Allows JS to manipulate the structure and style of the content.
    • Very important for Single Page Applications (SPA).
    • freeCodeCamp: What’s the Document Object Model, and why you should know how to use it.
      • Manipulate the DOM with events such as myEleName.addEventListener('click', function(event){}, false)
    • See structure even in this page
      • <html> → <body> → <div> → <p>
    • JS Tutorial: Document Object Model in JavaScript
  • Look through this: HTML5 Cheat Sheet

Questions

  • What is a proxy?
    • A proxy is a server between the user and the internet. It provides security as it has it's own IP address sent to external servers and only returns what was requested back to the user, while blocking anything unwanted.
    • The HTTP request is modified and handled by the proxy.
  • What is a URI?
    • URI stands for Uniform Resource Identifier
    • Uniform Resource Locator (URL) and Uniform Resource Name (URN) are parts of the URI.
  • What is CORS?
    • CORS stands for Cross-Origin Resource Sharing
    • Identifies which origins such as domain, schemes, or ports, that the browser should allow loading of resources.
