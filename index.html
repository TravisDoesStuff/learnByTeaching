  • What are the fundamentals of HTTP? Such as the header.
    • HTTP stands for HyperText Transfer Protocol
    • Request methods:
      • GET: retrieve information from server, parameters visible in URL
      • POST: sends information to server
      • PUT: same as POST but uses the same resource, for example when the browser refreshes, POST methods ask about resending the data
      • DELETE: removes the specific resource, infrequently used
      • HEAD: same as GET but without a response body, infrequently used
    • Headers are simply additional data sent in the request, often used for caching , authentication, and cookies.
    • Status Codes:
      • 1xx: Information
      • 2xx: Success
      • 3xx: Redirects
      • 4xx: Client error
      • 5xx: Server error
    • cURL is a HTTP command line tool.

    • A proxy is a server between the user and the internet. It provides security as it has it's own IP address sent to external servers and only returns what was requested back to the user, while blocking anything unwanted.
    • The HTTP request is modified and handled by the proxy.
  • What is a URI?
    • URI stands for Uniform Resource Identifier
    • Uniform Resource Locator (URL) and Uniform Resource Name (URN) are parts of the URI.
  • What is CORS?
    • CORS stands for Cross-Origin Resource Sharing
    • Identifies which origins such as domain, schemes, or ports, that the browser should allow loading of resources.

    • Positions: CSS-Tricks: position
      • Static
        • Default
        • Follows document flow.
        • Not affected by top, left, bottom, right, and z-index properties.
      • Relative
        • Follows document flow.
        • Positioned relative to itself on top, right, bottom, and left.
        • Usage is discouraged for positioning.
      • Fixed
        • Does not follow document flow.
        • Ignores element ancestry.
        • Fixed to screen position, even when scrolling.
      • Absolute
        • Does not follow document flow.
        • Placed relative to closest positioned parent container.
        • Fits to content.
        • If parent element is position: relative;, child element with position: absolute; will be set to parent.
      • Sticky
        • Follows user's scroll position
        • Combines relative and fixed positioning
        • Experimental

    • Displays, there are many but the most important: CSS-Tricks: display
      • inline
        • default
        • sits within content
        • padding and margin is allowed
      • block
        • breaks past inline
        • by default spans entire element
      • inline-block
        • similar to inline but allows width and height
      • flex
        • See Flexbox below
      • grid
        • See Grid below
      • none
        • Hides element
    • CSS Flexbox vs Grid, grid is apparently new.
      • Grid CSS-Tricks A Complete Guide to Grid
        • Is not new but is trending.
        • Where Flexbox is one-dimensional only caring about the container and all of it's content, Grid is two dimensional working with rows and columns.
        • Works VERY well with Flexbox as you can put a flex container in a grid cell.
        • Supports variable sizes.
      • Flex: CSS-Tricks: A Complete Guide to Flexbox
        • Allows flexability of container content when content is dynamic or unknown size.
        • Simply set parent element, or container, to display: flex;
        • Very useful for SPAs
        • Properties
          • Children order: x; property sets order of appearance in container.
          • flex-direction direction axis of content
          • flex-grow resizes content components to container
          • flex-wrap
            • nowrap default fit everything to one line
            • wrap wrap content top to bottom
            • wrap-reverse wrap content bottom to top
          • justify-content distributes whitespace between components horizontally
          • align-items distributes content with space top or bottom
          • align-content relevant to fixed container sizes, space around all inner content
      • property float by default has content wrap around it
      • Units:
        • px: absolute pixels, less useful for large screen sizes
        • em: relative to font size
        • rem: relative to root element
        • %: relative percentage to parent element

  • More arrow function goodness: argument => doSomethingAutoReturned
  • Rest operator myFunction(...arg){} creates an array of all passed in arguments in a function
    • Example: myFunction(1,2,3); function myFunction(...myArray); is equivalent to myArray = [1,2,3];
  • Spread operator when invoked within an array, will 'combine' the two together by 'spreading' the smaller array
    • Example: let smallArray = ['second', third']; let combinedArray = ['first', ...smallArray]; becomes ['first', 'second', 'third']
  • Object destructuring
    • Syntax: const { myObjectProperty : variableAssigment } = myObject;
    • Destructuring can be nested
    • Destructuring can also be used as arguments when invoking a function such as myFunction(myObj); myFunction({objProp1, objProp2}) {}
  • Array destructuring is very much like object destructuring but you cannot specify which index goes into a variable. Instead they go in order of indexes and assignments.
    • Empty commas [x, y, , , z] can be used as a workaround.
  • Backticks string literals support new lines without needing special characters.
  • Object literals are very much like prototypes in OOP.*
    • let createNewObject = (arg1, arg2, arg3) => ({prop1, prop2, prop3}); let myObj = createNewObject(arg1, arg2, arg3);
  • Objects can have functions of course.
  • Classes support constructors, getters, and setters in ES6.
    • _variables with underscores define a "private" variable.
    • get myGetterFunction() {} does not require parenthesis when invoked let myValue = myObject.myGetterFunction
    • set mySetterFunction() {} does not require parenthesis when invoked myObject.mySetterFunction = myValue
  • import { myFunctionName } from "./external_js_file" to use a function in another file exported by export const myFunctionName()
    • file looks for the magical words export to make it valid for importing
    • Export is also possible by export { myFunctionName, myVariableName } on its own line
    • You may import all like import * as myImportedObjectName from "./external_js_file";
    • export default ... serves as a fallback if an import fails
    • import myFunctionName from './external_js_file' without the curly brackets serves as the default import
  • myArray.filter() creates a new array based on a condition
